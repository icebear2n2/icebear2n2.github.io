CNCF(Cloud Native Computing Foundation)

2021-07-16 CNCF PV:


Cloud Native 정의

  • 클라우드 네이티브 기술을 통해 퍼블릭, 프라이빗 및 하이브리드 클라우드와 현대적이고 역동적인 환경에서 확장 가능한 애플리케이션을 구축하고 실행 가능
  • 컨테이너, Service Meshes, 마이크로 서비스, 불변적 인프라 및 선언적 API가 이 접근 방식에 예시

CNCF란 ?

Cloud Native Computing Foundation의 약자

컨테이너 기술의 발전과 그의 따른 기술 산업을 돕기 위해 2015년에 설립된 리눅스 재단(The Linux Foundation)의 프로젝트이다.
CNCF는 대표적으로 Kubernetes와 Prometheus 와 같은 클라우드 네이티브 오픈소스 기술들을 추진하고 관리하는 단체이다.

지금 컨테이너 세계는 다양한 기술이 섞여 있어 혼란스러운 상황에, 기업들은 컨테이너 환경에서 개발을 하려고 해도 어떻게 설계해야 할지 혼돈스러워하고 있다.

2016년에 출시한 CNCF가 혼란스러운 컨테이너와 관련된 다양한 기술적인 문제들을 오픈소스로 해결하는 것을 목표로 하고 있다.

CNCF는 2015년에 Linux Foudation 산하에 설립되어 첫 번쨰 프로젝트로 Google에서 Kubernetes를 기증하였다.

CNCF는 벤더 중립적인 오픈소스 프로젝트 생태계를 육성하고 유지함으로써 해당 패러다임 채택을 촉진한다.

Cloud Native의 4가지 요소

  • Microservice Architecture
  • DevOps
  • CI/ID
  • Container

CNCF 참여 기업

알리바바 클라우드, AWS, 애플, arm, CISCO, HUAWEI, INTEL, IBM, Microsoft Azure 등 .. 계속해서 증가하고 있다.
