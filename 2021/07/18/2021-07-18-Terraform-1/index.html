Infrastructure as Code (IaC)

코드를 통해 인프라를 관리하고 프로비저닝하는 것을 말한다.

IaC를 상요하면 구성을 편집하고 배포하기가 더 쉬워지고, 매번 동일한 환경을 프로비저닝하도록 보장한다.

IaC는 구성 사양을 코드화하고 문서화함으로써, 구성 변경 사항을 문서화하지 않고 임시로 변경하는 일을 막을 수 있다.

Terraform이란?

코드 도구로서의 HashiCorp의 인프라이다.

사용자가 읽을 수 있는 선언적 구성 파일에서 리소스와 인프라를 정의하고, 인프라의 수명 주기를 관리할 수 있다.

Terraform은 인프라애서 안전하게 협업하기 위해 구성을 버전 제어에 커밋할 수 있다.

Terraform으로 인프라를 배포하려면?

Scope

프로젝트의 인프라를 식별

Author

인프라에 대한 구성을 작성

Initialize

Terraform이 인프라를 관리하는 데 필요한 플러그인을 설치

Plan

구성과 일치하도록 Terraform이 수행할 변경 사항을 미리 본다

Apply

계획된 변경을 수행
