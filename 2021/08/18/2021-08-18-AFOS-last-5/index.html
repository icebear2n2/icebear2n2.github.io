[AWS] 10주차 -5 장애 발생 및 지속 서비스 확인

2021-08-18 AWS PV:

RDS DB Primary 장애

구성





RDS -> 작업 -> 재부팅

장애 조치 ~ 클릭 -> 확인 클릭

지켜봅시다.

한번 더 재부팅 시켜줍니다

IP주소가 4점대로 바뀌었습니다

정상적으로 DB 접근이 됩니다

리전은 a영역, 보조 영역은 c영역입니다

AZ1 장애

구성





WebSrv-Leader 인스턴스에서 지속적인 접속 확인

VPC -> 네트워크 ACL 클릭

네트워크 ACL 생성 클릭

네트워크 ACL 생성 클릭

작업 -> 서브넷 연결 편집 클릭

Subnet1, 3 선택 -> 변경 사항 저장 클릭

a 영역은 타겟그룹에서 제외되고 있습니다

부하분산되어지는 걸 확인하며 AutoScaling이 정상적으로 작동하는 걸 알 수 있습니다
